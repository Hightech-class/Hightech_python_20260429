### 자료구조
* 컴퓨터에서 자료를 정리하고 조직화하는 여러가지 구조
    * 자료마다 효율적인 정리 규칙이 존재
    * 단순 자료 구조(숫자, 문자)
    * 복합 자료 구조(여러 자료들을 한꺼번에 보관함(컨테이너))

* 복합 자료 구조
    *  선형(linear) 자료구조 : (자료들을 일렬로 나열(순서) / ex. 리스트, 스택, 큐, 덱)
    *  비선형(non-linear) 자료구조 : (한줄로 나열하기 어려운 자료들 / ex. 트리, 그래프)

### 알고리즘(algorithm)
* 문제를 해결하기 위한 절차/방법/명령어모음
    * 프로그램 = 자료구조 + 알고리즘
    * 프로그램 언어와는 무관

* 알고리즘 5대 성립 조건
    * 입력 : 0개 이상 존재(ex. 현재 시간 출력하는 알고리즘처럼 값을 아무것도 안넣어도 돌아가는 알고리즘이 존재함)
    * 출력 : 1개 이상 존재(왜? -> 알고리즘 정의에 따라 최소 **작업 완료**라는 결과라도 외부에 내보내야함. 왜냐하면 내가 실행했을때 출력이 0이면 이 알고리즘이 돌아가는지 안돌아가는지 알수가 없음)
    * 명백성 : 명령어의 의미는 **명확**해야함
    * 유한성 : 유한한 단계 후에는 반드시 종료해야함
    * 유효성 : 명령어들은 실행 가능한 연산이여야함
 
* 알고리즘 기술 방법
    * 자연어
    * 흐름도(flowchart) 
    * 유사코드(pseudo-code) : 알고리즘의 논리적인 절차를 사람이 이해하기 쉽게 일상 언어로 요약해서 쓴 것(ex. 요리법을 적어둔 레시피)

* 유사코드 예시
```
FUNCTION find_max(numbers):
    1. 변수 max_val에 리스트의 첫 번째 요소를 할당한다.
    2. 리스트의 두 번째 요소부터 끝까지 반복하며 다음을 수행한다:
        A. 만약 현재 요소가 max_val보다 크다면:
            - max_val을 현재 요소의 값으로 업데이트한다.
    3. 반복이 종료되면 최종 max_val을 반환한다.
END FUNCTION
```

* 유사코드를 파이썬으로 옮긴 코드
```
def find_max(numbers):
    # 1. 초기값 설정
    max_val = numbers[0]
    
    # 2. 반복 및 비교
    for num in numbers[1:]:
        if num > max_val:
            max_val = num
            
    # 3. 결과 반환
    return max_val

# 테스트 코드
data = [12, 45, 7, 23, 56, 10]
print(f"최댓값: {find_max(data)}") # 출력: 56
```

### 추상 자료형(Abstract Data Type, ADT)

* 추상적으로 정의한 자료형 : 데이터의 "무엇(What)"에 집중하고, 구체적인 "어떻게(How)"는 나중에 생각하겠다!
    * 핵심적인 **구조, 동작** 에만 집중
    * 제공되는 자료, 연산만 정의 (what)
    * 구현 방법은 정의x (how)

* 인터페이스만 공개
    * 정보은닉

### 알고리즘 성능 : 복잡도 분석(complexity analysis)

* 실제 구현 없이 알고리즘의 **효율성**분석
    * 알고리즘의 연산 횟수를 대략적으로 계산함
    * 하드웨어, 소프트웨어 환경에 무관한 평가 방법임(개별&연산의 속도 차이 무시)
    * 입력개수 n이 증가함에 따라 실행 시간&메모리의 증가형태를 분석함 

* 알고리즘을 구성하는 연산 횟수 분석
    * 시간복잡도 함수 T(n) : 연산의 수를 n의 함수로 표현 (데이터의 개수(n)에 따른 연산 횟수의 증가율을 봄) 
    * 공간복잡도 : 알고리즘이 실행되는동안 사용하는 메모리의 양을 측정함

### 리스트와 집합

* 리스트(list) / 선형 리스트(linear list) : 리스트는 **선형성**을 전제로 함(선형 : 데이터가 일렬로 나열된 상태)
    * 항목들이 차례대로 나열되어 있는 선형 자료
        * 항목은 순서/위치(position)를 가짐(데이터의 앞/뒤 관계가 뚜렷함)
    * 임의의 위치에 항목의 삽입 및 추가 가능
    * 리스트 vs 집합(set)
        * 항목들 사이의 순서 여부 
* LIST ADT
    * 연산시 최소 이 정도는 있어야함
        * Insert(pos ,e) : pos위치에 새로운 요소 e를 삽입
        * Delete(pos) : pos 위치에 있는 요소를 꺼내고(삭제) 반환함
        * isEmpty() : 리스트가 비어 있는지를 검사함
        * isFull() : 리스트가 가득 차 있는지를 검사함
        * getEntry(pos) : pos 위치에 있는 요소를 반환함

* 리스트 구현 방식
    * 배열(array) 구조
        * **같은** 자료형의 데이터를 **한꺼번에**만들 때 사용 
        * 인덱스 연산자 [] 사용함
        * 항목들이 메모리의 연속된 공간에 위치함
        * 용량 변경 어렵고, 배열의 중간에 대한 연산(데이터 추가&삭제&수정등)이 비효율적임 
    * 연결된 구조(linked structure)
        * 링크를 이용해 다음 항목의 **위치** 만 인지
        * **항목들이 메모리의 인접한 위치에 있다는 보장 불가** 
        * 용량 변경이 용이하고, 삽입&삭제가 효율적임 

* 리스트 구현 방법 **과제로 작성완료시 연결해서 수정**

### 스택

* 스택(stack) : **가장 나중에 들어온 데이터가 가장 먼저 나간다**
    * 후입선출(LIFO, Last-In First-Out) 형태의 자료구조
        * 스택 상단(top)으로만 삽입&삭제가 가능한 구조
        * 상단의 위치는 리스트의 맨 앞 or 맨 뒤를 사용  
    * 리스트에 비해 연산이 간단함
        * 입출력의 제한 
* 스택은 기본적으로 다음 4가지 동작을 수행함
    * Push: 데이터를 스택의 맨 위에 추가함
    * Pop: 스택의 맨 위에 있는 데이터를 꺼내서 반환함
    * Peek/Top: 맨 위에 있는 데이터를 꺼내지 않고 구경함
    * isEmpty: 스택이 비어 있는지 확인함

* 스택 활용 예시
    * 문서 편집기의 **되돌리기** 기능
    * 웹 브라우저의 **이전 페이지로 이동** 기능 
    * 시스템 스택
    * 소스코드 검사 코드
        * 괄호 닫기 
    * 계산기 프로그램
        * 수식 계산 과정 
    * 미로의 출구 찾기 

* 스택 구현 방법
    * 파이썬을 이용한 스택 ADT(오버플로(overflow), 언더플로(underflow)주의) 
        * 오버플로(최대크기를 넘어서는 현상)
        * 언더플로(가지고 올게 없는데 무언가를 가져오려는 현상)
* 스택 응용
    * 괄호 검사
        * 괄호갯수는 짝수여야함
        * 닫을때는 바로 앞에 여는 괄호가 있어야함
    * 예시
  ```
  def check_brackets(expression):
    stack = []
    # 괄호 매칭을 위한 딕셔너리
    mapping = {')': '(', '}': '{', ']': '['}

    for char in expression:
        if char in mapping.values():  # 여는 괄호인 경우
            stack.append(char)
        elif char in mapping.keys():  # 닫는 괄호인 경우
            if not stack or stack.pop() != mapping[char]:
                return False  # 짝이 맞지 않거나 스택이 빈 경우(언더플로 상황)
                
    return len(stack) == 0  # 마지막에 스택이 비어 있어야 True

    # 테스트
      print(check_brackets("({[]})"))  # True
      print(check_brackets("({[})"))    # False
    ```
  
* 스택 응용
    * 수식 계산 : 우리는 중위표기법(A+B)에 익숙하지만 컴퓨터는 후위표기법(AB+)을 좋아함
        * 연산자 표기법엔 전위(+AB),중위(A+B),후위(AB+) 3가지 표기법이 있음   
    * 컴퓨터가 후위 표기법(Postfix)을 쓰는이유
        * 괄호가 필요 없음 : 연산 순서가 이미 배치 순서에 결정되어 있어 괄호를 쓰지 않아도 혼동이 없음
        * 우선순위 고민x : 앞에서부터 차례대로 읽으면서 스택에 넣고 빼기만 하면 계산이 끝남
        * 컴퓨터 친화적 : 컴파일러가 수식을 처리할 때 가장 효율적인 방식
    * 중위연산자->후위연산자로 바꿔주는 로직
      ```
         def infix_to_postfix(expression):
       # 우선순위 설정
       prec = {'*': 3, '/': 3, '+': 2, '-': 2, '(': 1}
       stack = []
       result = []
    
       for char in expression:
           # 1. 숫자/문자인 경우
           if char.isalnum():
               result.append(char)
           # 2. 여는 괄호인 경우
           elif char == '(':
               stack.append(char)
           # 3. 닫는 괄호인 경우
           elif char == ')':
               while stack and stack[-1] != '(':
                   result.append(stack.pop())
               stack.pop() # '(' 제거
           # 4. 연산자인 경우
           else:
               while stack and prec[stack[-1]] >= prec[char]:
                   result.append(stack.pop())
               stack.append(char)
               
       # 나머지 털어내기
       while stack:
           result.append(stack.pop())
           
       return "".join(result)
   
         # 테스트
         print(infix_to_postfix("(A+B)*C")) # 출력: AB+C*
      ```

### 고급 탐색 알고리즘

* 해싱(hashing) : 간단한 탐색 알고리즘은 비교하는 과정을 통해 찾음
    * 일어날 수 있는 문제점
        * 충돌(collision) : 서로 다른 키가 해시함수에 의해 같은 주소로 계산되는 상황
            * 해결방법
                * 체이닝(chaining) : 같은 주소로 배정된 데이터들을 연결리스트로 줄 세움
                * 개방주소법(open addressing) : 충돌이 나면 다른 리스트의 공간을 찾아나감
        * 오버플로(oveflow) : 충돌이 슬롯 수보다 더 많이 발생
            * 해결방법
                * 선형조사법(linear probing)으로 처리
                    * 선형조사법 : 해싱 함수로 계산된 버킷에 빈 슬롯이 없으면 그 다음 버킷에서 빈 슬롯을 찾아나가는 방법    
      
    * 탐색연산 : 해시 주소에 의해 레코드를 찾는 과정
    * 삭제연산 : 선형조사법에서 항목이 삭제되면 조사가 불가능해 질 수 있음 -> 조사 연결 고리(Probe Chain)'가 끊어지기 때문
    * 군집화 완화 방법 : 이차 조사법, 이중해싱법
        * 이차 조사법[작성필요]
        * 이중 해싱법[작성필요]       
    
    * 좋은 해시 함수가 되려면 -> 충돌이 적음,계산이 빠름, 생성되는 주소가 테이블에서 고르게 분포
    * 해시 함수 예
        * 폴딩(folding) 함수 : 탐색기가 해시 테이블의 크기보다 더 큰 정수일때 사용
        * 중간제곱함수
        * 비트추출함수
        * 숫자분석방법 
        * 탐색키가 문자열인 경우 **문자를 정수로 대응**하여 사용 

    * 탐색 방벙 성능
        * 적재밀도/적재비율

### 트리

* 트리 : 계층적인 자료(직장 조직도,가계도 등)의 표현에 적합한 자료 구조
    * 활용 방안
        * 탐색 트리(탐색)
        * 힙 트리(우선순위 큐)
        * 결정 트리(의사결정 구조) 

    * 용어
        * 루트 노드 : 계층적인 구조에서 **가장 높은 곳**에 있는 노드
        * 간선 / 에지 : 노드와 노드를 연결하는 선
        * 부모, 자식노드 : 직접 연결된 노드중에 상,하위 노드
        * 형제 노드 : 동일한 부모노드를 가진 노드
        * 조상,자손 노드
